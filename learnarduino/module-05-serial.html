Serial Communication — Talking to Your Arduino
The serial monitor is your window into what your Arduino is thinking. Learn to send data out, read commands in, and use serial as a debugging superpower.
LESSONThe Concepts

Serial Communication

Serial is how your Arduino talks to your computer. It sends data one bit at a time over a wire — that's why it's called "serial" (one after another).

Setting Up Serial

void setup() {
  Serial.begin(9600);  // Start serial at 9600 baud
}

9600 is the baud rate — how fast data is sent. Both sides must agree on the speed.

Sending Data Out

Serial.println("Hello!");   // Print with newline
Serial.print("No newline"); // Print without newline
Serial.println(42);         // Print numbers too

Reading Data In

if (Serial.available() > 0) {
  char c = Serial.read();  // Read one byte
  // c now contains what was typed
}

Serial.available() returns how many bytes are waiting to be read. Serial.read() reads one byte and removes it from the buffer.

Your Task

Build a serial-controlled LED. Type 1 in the serial input to turn the LED on, 0 to turn it off. Print the current state back to the monitor.

Debugging tip: Sprinkle Serial.println() throughout your code to see what's happening. It's the easiest way to find bugs.
