State Machines — Thinking in States
Real systems have modes — a traffic light is red, then green, then yellow. State machines give you a clean pattern to manage these transitions without messy nested ifs.
LESSONThe Concepts

What Is a State Machine?

A state machine is a design pattern where your program is always in one defined state, and transitions between states based on conditions (time, input, sensor data).

Defining States with enum

enum State {
  RED,
  YELLOW,
  GREEN
};

State currentState = RED;

An enum creates named constants — much clearer than using magic numbers like 0, 1, 2.

The switch/case Pattern

switch (currentState) {
  case RED:
    // do red things
    break;
  case YELLOW:
    // do yellow things
    break;
  case GREEN:
    // do green things
    break;
}

millis() vs delay()

delay() blocks everything — nothing else can happen. millis() returns time since boot, letting you check elapsed time without blocking:

unsigned long prevTime = 0;
unsigned long interval = 5000; // 5 seconds

void loop() {
  if (millis() - prevTime >= interval) {
    prevTime = millis();
    // Time to transition!
  }
  // Other code still runs here
}

Your Task

Build a traffic light that cycles RED (5s) → GREEN (4s) → YELLOW (2s) → RED using millis() and switch/case. Watch the LEDs change in the simulator.

Why millis()? With delay(), your Arduino is frozen. With millis(), it can still read buttons, update displays, and respond to events while waiting.
