LESSONProject Guide

Design Your Project

A good project combines at least 3 concepts from previous modules. Think about what you want your Arduino to do, then work backwards to figure out how.

Project Planning Checklist

  • Purpose: What problem does your project solve? What does it do?
  • Inputs: What sensors, buttons, or data does it read?
  • Processing: What logic, states, or calculations does it perform?
  • Outputs: What LEDs, servos, LCD text, or serial data does it produce?
  • States: Does your project have different modes? Draw a state diagram.

Project Ideas

  • Distance alarm — ultrasonic sensor + LED + buzzer pattern
  • Temperature display — DHT11 + LCD + warning LEDs
  • Automated gate — ultrasonic sensor + servo + state machine
  • Light-tracking servo — LDR + servo + serial logging
  • Traffic light with pedestrian button — state machine + button + LEDs

Documentation Tips

  • Comment every section of your code explaining why, not just what
  • Use meaningful variable names: distanceCm not d
  • Group related code into functions
  • Print debug info to Serial during development
